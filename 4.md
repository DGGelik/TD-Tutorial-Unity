# План разработки: Crystal Bastion  
**Детальный план по шагам — Месяц 4**

**Цель месяца:** Финальная полировка, удобный интерфейс, меню, тестирование, сборка и презентация проекта.  
**Результат к концу месяца:** Готовая игра, которую можно показать родителям/учителям/друзьям, запустить на любом ПК, выложить на GitHub + itch.io, записать короткое видео прохождения.  
**Время на месяц:** 8–10 занятий + много самостоятельного тестирования и доработки.  
**Предполагаем:** Месяц 3 завершён — все механики работают, эффекты и звук на месте, 12 волн сбалансированы.


## Неделя 13: Главное меню, пауза и навигация

### День 37: Главное меню
**Что делаем:**
```
1. File → New Scene → MainMenu
2. Canvas → Scaler: Scale With Screen Size
3. Элементы:
   - Логотип / название игры (большой текст + фоновая картинка замка)
   - Кнопки:
     • Play → LoadScene("MainScene")
     • Options (пока пустой)
     • Quit
   
4. BackgroundMusic → отдельный префаб, DontDestroyOnLoad
```

**Результат:** При запуске проекта открывается красивое меню.

### День 38: Пауза и игровое меню
**Что делаем:**
```
1. В MainScene → Canvas → PausePanel (неактивен по умолчанию)
   - Кнопки: Resume, Restart, Main Menu, Quit
   
2. Input: Esc → PausePanel.SetActive(true) + Time.timeScale = 0
   
3. Resume → PausePanel.SetActive(false) + Time.timeScale = 1
```

**Результат:** Полноценная пауза во время игры.

### День 39: Настройки (простые)
**Что делаем:**
```
1. OptionsPanel в MainMenu:
   - Slider: Master Volume
   - Slider: Music Volume
   - Slider: SFX Volume
   
2. AudioMixer → exposed parameters: MasterVol, MusicVol, SFXVol
3. PlayerPrefs для сохранения громкости
```

**Результат:** Игрок может регулировать звук.

**Контрольная точка недели 13:** Меню, пауза и базовые настройки работают ✅

---

## Неделя 14: Финальная визуальная полировка

### День 40: Экран загрузки / переходы
**Что делаем:**
```
1. Новая сцена LoadingScreen
2. Анимация: вращающийся кристалл или прогресс-бар
3. SceneManager.LoadSceneAsync → обновлять прогресс-бар
```

### День 41: Улучшение UI и анимации
**Что делаем:**
```
1. GoldText: анимация +1 (прыжок вверх при получении)
2. Damage numbers: Canvas World Space над врагом при уроне
3. VictoryPanel: конфетти ParticleSystem + fade-in
4. GameOverPanel: драматичный звук + fade-out
```

### День 42: Камера и пост-эффекты
**Что делаем:**
```
1. Cinemachine → Virtual Camera (Follow + LookAt на центр карты)
2. Post Processing (URP):
   - Bloom на выстрелах и боссе
   - Vignette на поздних волнах
   - Color Grading: тёплые тона на победу, холодные на поражение
```

**Контрольная точка недели 14:** Игра выглядит гораздо красивее и профессиональнее ✅

## Неделя 15: Тестирование, баги и оптимизация

### День 43: Полное тестирование
**Что делаем:**
```
1. Checklist на 30–40 пунктов:
   - Все башни улучшаются до 3 уровня?
   - Продажа работает во время волны?
   - Звуки не накладываются?
   - Призраки игнорируются стрелковыми башнями?
   - Босс появляется только на 12-й волне?
   - Нет лагов при 30+ врагах?
   
2. Каждый ученик проходит уровень 3–5 раз и записывает баги
```

### День 44: Исправление критических багов
**Что делаем:**
```
1. Object Pooling для пуль и эффектов (если ещё не сделано)
2. Fix: башни стреляют после смерти врага → проверка target != null
3. Fix: UI не обновляется при продаже
4. Fix: музыка не переключается
```

### День 45: Финальная оптимизация
**Что делаем:**
```
1. Profiler → искать пики CPU/GPU
2. Уменьшить количество частиц при необходимости
3. Build Settings → Player Settings:
   - Resolution: Fullscreen Window
   - Graphics: Low / Medium
```

**Контрольная точка недели 15:** Игра стабильна, FPS 60+ даже на слабых ПК ✅
